歩数マップ作成（幅優先探索）

関数名 

calc_map

引数 

No. シンボル名 型 説明 値域
1. - void
 - -

戻り値 

名前 型 値域
- void
 -

処理概要 

迷路のマップ壁情報とゴール位置をもとに、ゴールまでの歩数マップを作成する。

ゴールを0歩で行けるマスとし、次に1歩で行けるマスを全て探し、次に2歩で行けるマスを全て探し、と繰り返すことで、歩数マップを全て完成させる。

 

歩数マップは以下のように求めていく。

  1. 1歩で行けるマスを全て探す
  2. 1歩で行けるマスからさらに1歩で行けるマスを全て探す（つまり2歩で行ける）
  3. 2歩で行けるマスからさらに1歩で行けるマスを全て探す（つまり3歩で行ける）
  4. 以降繰り返し

 

 

処理の流れは以下のようにする。

処理仕様 

No. 内容
1. 歩数マップにおける全ての座標の歩数に、未到達扱いにするため255を設定する。
2. 歩数マップにおけるゴール位置の歩数を0とする。
3. 0歩から36歩までを1歩ずつ順番に対象歩数とし、No.4以降の処理を実施する。
4. 歩数マップ全体から、対象歩数のマスを探し、見つかったマスに対してNo.5~8を実施する。
5. 見つかったマスのマップ壁情報と歩数マップを確認し、北に壁がなく、北に隣接するマスの歩数が見つかったマスの歩数+1より大きければ、
北に隣接するマスの歩数マップの歩数を 見つかったマスの歩数+1 とする。（例：step_map[x][y+1] = stepmap[x][y] + 1）
6. 見つかったマスのマップ壁情報と歩数マップを確認し、東に壁がなく、東に隣接するマスの歩数が見つかったマスの歩数+1より大きければ、
東に隣接するマスの歩数マップの歩数を 見つかったマスの歩数+1 とする。（例：step_map[x+1][y] = stepmap[x][y] + 1）
7. 見つかったマスのマップ壁情報と歩数マップを確認し、南に壁がなく、南に隣接するマスの歩数が見つかったマスの歩数+1より大きければ、
南に隣接するマスの歩数マップの歩数を 見つかったマスの歩数+1 とする。（例：step_map[x][y-1] = stepmap[x][y] + 1）
8. 見つかったマスのマップ壁情報と歩数マップを確認し、西に壁がなく、西に隣接するマスの歩数が見つかったマスの歩数+1より大きければ、
西に隣接するマスの歩数マップの歩数を 見つかったマスの歩数+1 とする。（例：step_map[x-1][y] = stepmap[x][y] + 1）
