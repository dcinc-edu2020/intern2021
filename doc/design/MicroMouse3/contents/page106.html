隣接マス探索

関数名 

search_next_cell

引数 

No. シンボル名 型 説明 値域
1. x int
 探索対象マスのx座標 0 - 5
2. y int
 探索対象マスのy座標 0 - 5

戻り値 

名前 型 値域
- void
 -

処理概要 

引数で指定された探索対象マスから、上下左右4方向に隣接するマスに対し、

壁の有無と歩数マップの歩数から、その方向に探索を進めるべきかを判断し、

探索を進める場合はそのマスを探索対象マスとして隣接マス探索を呼び出す。

 

隣接マスへの探索は、以下2つの条件をどちらも満たすときその方向に探索を進めるべきと判断する。

・間に壁がない

・隣接マスはまだ探索していない、もしくは、探索済みだが今回のほうが歩数が少ない

　－未探索のマスは歩数が255になっているため、今回の歩数のほうが少ないかどうかのみを確認することで両方の条件が確認できる。

 

処理の流れは以下のようにする。

処理仕様 

No. 内容
1. 探索対象マスのマップ壁情報と歩数マップを確認し、北に壁がなく、北に隣接するマスの歩数が探索対象マスの歩数+1より大きければ、以下を実施。
・北に隣接するマスの歩数マップの歩数を 探索対象マスの歩数+1 とする。（例：step_map[x][y+1] = stepmap[x][y] + 1）
・北に隣接するマスを引数（探索対象マス）とし、「隣接マス探索」を呼び出す。（例：search_next_cell(x,y+1)）
2. 探索対象マスのマップ壁情報と歩数マップを確認し、東に壁がなく、東に隣接するマスの歩数が探索対象マスの歩数+1より大きければ、以下を実施。
・東に隣接するマスの歩数マップの歩数を 探索対象マスの歩数+1 とする。（例：step_map[x+1][y] = stepmap[x][y] + 1）
・東に隣接するマスを引数（探索対象マス）とし、「隣接マス探索」を呼び出す。（例：search_next_cell(x+1,y)）
3. 探索対象マスのマップ壁情報と歩数マップを確認し、南に壁がなく、南に隣接するマスの歩数が探索対象マスの歩数+1より大きければ、以下を実施。
・南に隣接するマスの歩数マップの歩数を 探索対象マスの歩数+1 とする。（例：step_map[x][y-1] = stepmap[x][y] + 1）
・南に隣接するマスを引数（探索対象マス）とし、「隣接マス探索」を呼び出す。（例：search_next_cell(x,y-1)）
4. 探索対象マスのマップ壁情報と歩数マップを確認し、西に壁がなく、西に隣接するマスの歩数が探索対象マスの歩数+1より大きければ、以下を実施。
・西に隣接するマスの歩数マップの歩数を 探索対象マスの歩数+1 とする。（例：step_map[x-1][y] = stepmap[x][y] + 1）
・西に隣接するマスを引数（探索対象マス）とし、「隣接マス探索」を呼び出す。（例：search_next_cell(x-1,y)）
